# ServerSaver Export Specification

This document is intended for any future agent or developer building an **importer** for the Harmony platform. It details the exact structure, schema, and capabilities of the export packages generated by the `ServerSaver` Discord bot.

## Overview
ServerSaver connects to a Discord server, iterates through its history, and creates a comprehensive, static backup of the server's state, channels, members, messages, media, reactions, and custom emojis. The exporter has been specifically architected to be ingested natively by Harmony. 

## Directory Architecture
The export is fully self-contained. No external CDN links are relied upon for attachments; everything is downloaded locally. Directories and filenames are sanitized to ensure cross-OS compatibility (removing illegal characters).

### 1. Root Level: Server Context
**File:** `guild_metadata.json`

Provides the global state of the server. 
**Schema:**
- `id` (int64): Unique Discord Guild ID.
- `name` (string): Text display name.
- `owner_id` (int64)
- `description` (string | null)
- `icon_url` (string | null)
- `roles` (array):
  - `id` (int64)
  - `name` (string)
  - `color` (string: Hex code e.g. "#RRGGBB")
  - `position` (int): Hierarchy tier.
  - `permissions` (int64): Bitfield of base permissions.
- `members` (array): User node graph.
  - `id` (int64)
  - `name` (string): Username.
  - `global_name` (string | null): Display name.
  - `nickname` (string | null): Server-specific nickname.
  - `bot` (boolean)
  - `roles` (array of int64): IDs of roles applied to this member.
  - `avatar_url` (string | null): Global profile picture URL.
  - `server_avatar_url` (string | null): Server-specific profile picture URL.
  - `joined_at` (string | null): ISO 8601 UTC timestamp of when they joined the server.
  - `premium_since` (string | null): ISO 8601 UTC timestamp of when they boosted the server.
- `emojis` (array): Custom server emotes.
  - `id` (int64)
  - `name` (string)
  - `url` (string): Remote URL for fallback.
  - `animated` (boolean)
- `categories` (array): Channel groupings.
  - `id` (int64)
  - `name` (string)
  - `position` (int)

---

### 2. Channel Level: Metadata
**File:** `[Channel Name]-[Channel ID]/channel_metadata.json`

Provides context about the physical space.
**Schema:**
- `id` (int64)
- `name` (string)
- `topic` (string | null)
- `position` (int): Position in the channel list.
- `category_id` (int64 | null): The ID of the category group it belongs to, if any.
- `nsfw` (boolean)

---

### 3. Channel Level: Messages
**File:** `[Channel Name]-[Channel ID]/messages.json`

An array of **chronologically ordered** (oldest to newest) message objects. 

**Schema:**
- `id` (int64): Unique Message ID.
- `timestamp` (string): ISO 8601 UTC timestamp (e.g., `2023-10-27T12:00:00+00:00`).
- `author_id` (int64): The ID of the user, cross-referenced with `guild_metadata.members`.
- `author_name` (string): Fallback identifier (Username or Username#Discrim).
- `content` (string): The raw text (markdown).
- `attachments` (array of strings): **IMPORTANT:** These are *local filenames* relative to the `media/` folder. (e.g. `"12345_image.png"` refers to `./media/12345_image.png`). If download failed, it may fallback to an external HTTP URL.
- `embeds` (array of objects): Standard Discord embed dictionary objects.
- `is_pinned` (boolean)
- `reactions` (array):
  - `emoji` (string): Standard unicode emoji (e.g., "đź‘Ť") or the name of a custom emoji.
  - `count` (int): Total count.
  - `users` (array, optional): Present if `record_reaction_users` is enabled.
    - `id` (int64): Unique Discord User ID.
    - `name` (string): Username (the `str(user)` representation, which may include a discriminator for older accounts).

---

### 4. Channel Level: Media Folder
**Directory:** `[Channel Name]-[Channel ID]/media/`

Contains all binary files (images, videos, etc.) downloaded. 
**Naming convention:** `[Message ID]_[Sanitized Filename]` to ensure uniqueness and auditability.

## Technical Notes
1. **Precision Warning:** Discord IDs are 64-bit integers. Since standard JSON parsers in languages like JavaScript may lose precision (IEEE 754), importers MUST treat these IDs as **strings** during parsing. (The `ServerSaver` bot writes them as literal numbers, so a streaming parser with string conversion is recommended).
2. **Incremental Logic:** `ServerSaver` is designed to be resumeable. If `messages.json` already exists, it reads the last message ID and fetches only messages *after* that point. An importer should similarly be able to handle incremental updates to the same channel.
3. **Encoding:** All JSON files are encoded in **UTF-8** with `ensure_ascii=False`.
4. **Resiliency:** If a media download fails, `ServerSaver` leaves the original URL in the `attachments` array. Importers must handle both local file paths and remote URLs.
5. **API Stability:** The bot includes built-in exponential backoff (retrying up to 5 times) for Discord API service errors (HTTP 500, 502, 503, 504). If the bot still fails after retries, it will save the progress made so far and proceed to the next channel.
6. **Reaction Logic:** If the bot lacks access to a custom emoji used in a reaction (e.g., from a different server, or deleted), that specific reaction may be skipped or have its user list truncated to prevent export failure.

## Importer Guidelines
1. **Relational Ingestion:** Ingest `guild_metadata.json` first to build the Server, Roles, and Profiles tables.
2. **Channel Hydration:** Build the channel map grouping by `category_id`, sorting by `position`.
3. **Media Hosting:** Move files from `media/` to Harmony's internal storage and update message records with the new URLs.
4. **Sanitization:** Use the `[Channel Name]-[Channel ID]` directory naming to resolve channel IDs even if names have changed.
